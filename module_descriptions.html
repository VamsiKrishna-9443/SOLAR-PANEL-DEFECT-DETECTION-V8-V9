MODULE DESCRIPTIONS

Solar Panel Defect Detection System — YOLOv8

1

User Module

Handles all end-user interactions with the system.

Key Features:

  • Registration & Login — Users register and sign in via Clerk authentication (Google OAuth / email-password).
  • Image Upload — Upload solar panel images (JPG/PNG) for defect analysis.
  • View Predictions — See AI-detected defects with annotated result images, defect counts, and type breakdown.
  • Detection History — Browse, review, and delete past detection results.
  • Profile Management — View and manage profile via Clerk’s UserButton component.
  • Theme Switching — Toggle between Light and Dark modes.

Frontend: Home.jsx, Detection.jsx, History.jsx, ThemeToggle.jsx

Backend: POST /predict/yolov8, GET /history/{user_id}, POST /history/save

2

Admin Module

Provides administrative control over the system and its users.

Key Features:

  • Admin Dashboard — A dedicated dashboard accessible only to users with the admin role.
  • User Management — View all registered users with name, email, join date, role, and access status.
  • Access Control — Grant or revoke detection access for individual users with a single click.
  • Statistics Overview — View total number of users and the count of users with active access.
  • Clerk Metadata Sync — Permission updates are synced back to Clerk’s publicMetadata via the Backend API.

Frontend: AdminDashboard.jsx

Backend: GET /admin/users, POST /admin/grant-access/{clerk_id}, POST /admin/sync-user

3

Authentication Module

Manages secure user authentication and session handling across the application.

Key Features:

  • Clerk Integration — Uses Clerk as the third-party authentication provider for sign-up, sign-in, and sessions.
  • OAuth Support — Supports social login via Google.
  • Role-Based Access Control (RBAC) — Distinguishes between user and admin roles using Clerk’s publicMetadata.
  • Protected Routes — Routes (/detect, /history, /admin) are guarded; unauthenticated users see “Sign In Required”.
  • Webhook Verification — Backend verifies Clerk webhook payloads using Svix for secure user sync events.

Frontend: main.jsx (ClerkProvider), App.jsx (SignedIn/SignedOut guards)

Backend: POST /webhooks/clerk, sync.py

4

Detection / Prediction Module

Core AI module that performs defect detection on uploaded solar panel images.

Key Features:

  • Image Processing — Accepts uploaded images, saves them to the server, and feeds them to the YOLOv8m model.
  • YOLOv8m Inference — Runs object detection using a YOLOv8m model trained on 12 solar panel defect classes.
  • Result Generation — Produces an annotated image with bounding boxes drawn around detected defects.
  • Defect Classification — Identifies and counts defects across 12 classes:
1. black_core2. corner3. crack4. finger 5. fragment6. horizontal_dislocation7. printing_error8. scratch 9. short_circuit10. star_crack11. hick_line12. vertical_dislocation

Backend: predictor.py, model_loader.py, POST /predict/yolov8

5

History Module

Stores, retrieves, and manages past detection results for each user.

Key Features:

  • Save Results — After each prediction, the detection result (image URL, defects, metadata) is saved to MongoDB.
  • View History — Users can browse past scans in a list view, sorted newest first.
  • Detail View — Click any past scan to see the full annotated result image and defect breakdown.
  • Delete Scans — Users can delete individual detection records or clear entire history.

Frontend: History.jsx

Backend: detections.py, POST /history/save, GET /history/{user_id}, DELETE /history/{user_id}

6

Database Module

Handles all data persistence and database connectivity for the system.

Key Features:

  • MongoDB Atlas — Cloud-hosted NoSQL database for storing all application data.
  • Motor (Async Driver) — Uses the motor async driver for non-blocking database operations with FastAPI.
  • Users Collection — Stores synced user profiles (clerkUserId, name, email, role, hasAccess, timestamps).
  • Detections Collection — Stores detection history records (clerkUserId, fileName, defects, result_image_url, timestamp).
  • CRUD Operations — Full Create, Read, Update, Delete support for both collections.

Backend: db.py, detections.py, sync.py
